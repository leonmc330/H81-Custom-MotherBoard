================================================================================
Two files in this project carry code derived from dhuertas/DES
(https://github.com/dhuertas/DES):

  1. xzzpcb_parser.py — pure-Python port of the DES algorithm
     (used as the fallback path for XZZPCB part-record decryption when
     xzz_native.dll isn't available).

  2. xzz_native.c — C port of the same DES algorithm
     (compiled into xzz_native.dll for the fast path; ~100× faster than
     the pure-Python fallback on a typical motherboard's encrypted records).

Both files port the same upstream reference implementation. That C
implementation is also reproduced in OpenBoardView under
src/openboardview/Crypto/des.c, where it carries the following header:

    /*
     * Data Encryption Standard
     * An approach to DES algorithm
     *
     * By: Daniel Huertas Gonzalez
     * Email: huertas.dani@gmail.com
     * Version: 0.1
     *
     * Based on the document FIPS PUB 46-3
     *
     * Source: https://github.com/dhuertas/DES
     */
